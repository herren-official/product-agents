# Commit Message Suggestion

추가되거나 수정 중인 파일을 분석하고, 한국어 커밋 메시지를 한 줄로 추천한다.
브랜치명은 작업 의도 파악의 단서로 활용한다.

## 트리거

- `/commit`
- "커밋 메시지 추천", "커밋 메시지 만들어줘", "commit message"

## 워크플로우

### Step 1: 변경사항 수집

```bash
git status --short
git diff --staged
git diff
```

- staged + unstaged 파일을 모두 확인
- 변경 라인을 파악하여 작업의 본질을 추출

### Step 2: 브랜치명 컨텍스트 활용

```bash
git branch --show-current
```

- 브랜치명에서 GBIZ 번호 / 키워드 추출
- 예: `GBIZ-26248-move-post-payment` → "결제 후처리 이동"이 맥락

### Step 3: 커밋 메시지 1줄 추천

다음 원칙에 따라 한국어 1줄로 작성:

1. **간결성**: 한 문장, 50자 내외
2. **동사형**: "추가", "수정", "삭제", "리팩터링", "개선" 등
3. **What 중심**: 무엇이 바뀌었는지 (Why는 PR 본문에)
4. **브랜치 의도 반영**: 브랜치명이 가리키는 목적과 일치

### Step 4: 사용자에게 제시

```
추천 커밋 메시지:
> {제안 메시지}

이대로 커밋할까요? 수정이 필요하면 알려주세요.
```

승인 시:
```bash
git commit -m "{메시지}"
```

## 출력 예시

| 변경 내용 | 추천 메시지 |
|----------|------------|
| 네이버 결제 핸들러 신규 추가 | 네이버 스토어 결제 핸들러 추가 |
| Repository 메서드 시그니처 변경 + 호출부 동기화 | 결제 Repository 시그니처 변경에 따른 호출부 정리 |
| 테스트 케이스만 보강 | 결제 후처리 통합 테스트 보강 |

## 주의사항

- 한국어로만 작성 (영문 prefix 제외)
- 커밋 자체를 자동 실행하지 않고, 메시지 제안 후 사용자 승인 받는다
- staged 가 비어있으면 `git add` 대상도 함께 제안
