# Create PR

현재 브랜치의 변경사항을 분석하고, 공비즈 CRM 프로젝트 PR 템플릿에 맞춰 GitHub PR을 생성한다.

## 트리거

- `/create-pr`
- `/create-pr {백로그링크}`
- `/create-pr {백로그링크} {베이스브랜치}`
- "PR 만들어줘", "PR 올려줘"

## 입력 인자

- `$1`: 백로그 링크 (선택, 미제공 시 빈 링크로 처리)
- `$2`: 베이스 브랜치 (선택, 미제공 시 `develop` 사용)

## PR 제목 형식

```
[CRM][GBIZ-xxxxx]: 내용
```

- 브랜치명에서 `GBIZ-xxxxx` 번호를 추출하여 사용
- '내용' 부분은 커밋 메시지를 분석하여 주요 작업 내용을 요약
- 예: 브랜치 `GBIZ-22343-naver-store-payment` + 결제 관련 커밋 → `[CRM][GBIZ-22343]: 네이버 스토어 결제 기능 추가`

## 워크플로우

### Step 1: 변경사항 분석

1. 현재 브랜치 확인
2. 베이스 브랜치 결정:
   - `$2` 가 제공된 경우: 해당 값 사용
   - `$2` 가 비어있는 경우: `develop` 사용
3. `git log {베이스브랜치}..HEAD` 로 커밋 히스토리 확인
4. `git diff {베이스브랜치}...HEAD --stat` 로 변경 파일 목록 확인

### Step 2: PR 본문 작성

PR 템플릿 형식에 맞춰 본문 작성:

```
## 개요
(이 PR의 목적과 배경을 간략히 설명)

## 작업사항
(구체적인 작업 내용을 bullet point로 나열)

## 변경로직

### 변경전
(기존 동작 방식 설명)

### 변경후
(변경된 동작 방식 설명)

## 사용방법
(필요한 경우 사용 방법 설명)

## 기타
(리뷰어가 알아야 할 추가 정보)

## 참고
* [GBIZ-xxxxx](백로그링크)
```

- `GBIZ-xxxxx`: 브랜치명에서 추출
- 백로그 링크: `$1` 이 있으면 사용, 없으면 빈 괄호 `()`

### Step 3: PR 생성 명령 실행

```bash
gh pr create --base {베이스브랜치} --title "PR제목" --body "$(cat <<'EOF'
{본문 내용}
EOF
)"
```

- HEREDOC을 사용해 본문을 안전하게 전달
- 줄바꿈, 특수문자 보존

## 주의사항

- 베이스 브랜치 결정 규칙을 반드시 지킨다 ($2 우선, 미제공 시 develop)
- HEREDOC 사용 필수 (본문에 특수문자/마크다운 포함되므로)
- 브랜치명에 `GBIZ-` 패턴이 없으면 사용자에게 확인
