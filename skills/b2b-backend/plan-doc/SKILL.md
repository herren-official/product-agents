현재 대화에서 수립된 구현 계획을 정리하여 문서로 저장해라.

## 절차

### Step 1: 브랜치명 조회 및 저장 경로 생성

- 프로젝트 루트는 `git rev-parse --show-toplevel`로 조회한다.
- `git branch --show-current`로 현재 브랜치명을 가져온다.
- 브랜치명에 `/`가 포함되면 `-`로 치환한다. (예: `feature/order-cancel` → `feature-order-cancel`)
- 저장 경로: `{프로젝트 루트}/.claude/plan/{branch-name}-plan.md`
- `.claude/plan/` 디렉토리가 없으면 생성한다.
- 동일 파일명이 이미 존재하면 사용자에게 덮어쓰기 여부를 확인한다.
- 브랜치명 조회에 실패하면 (detached HEAD 등) 사용자에게 파일명을 직접 지정하도록 요청하고 중단한다.

### Step 2: 현재 대화에서 계획 수집

현재 대화의 전체 메시지(사용자 입력 및 Claude 응답 모두)에서 다음 정보를 수집한다:
- `/plan` 모드 또는 자연어 대화에서 수립된 구현 계획 내용
- 계획 수립 이후의 질의응답에서 확정되거나 변경된 사항
- 사용자가 명시적으로 결정한 기술적 선택 (언어, 라이브러리, 설계 방식 등)

수집할 수 없는 항목은 "[미결정]"으로 표기한다.

**필수 섹션**: "목표", "결정된 구현 방향", "구현 단계"는 반드시 포함한다. 대화에서 해당 내용을 추출할 수 없으면 "[미결정]"으로 표기한다.

**선택 섹션**: "배경", "영향 범위", "하위 호환성", "테스트 전략", "롤백 시나리오", "마이그레이션 전략"은 `/plan` 모드 출력 또는 대화에서 해당 내용이 도출되었으면 포함한다. 양쪽 모두에서 언급되지 않은 섹션만 생략한다.

### Step 3: 문서 작성 및 저장

아래 형식으로 문서를 작성하여 저장한다. 제목은 대화에서 합의된 작업 목표를 한 줄로 요약한다. 명시적 제목이 없으면 브랜치명을 기반으로 자연어 제목을 생성한다.

`generated-at` 시각은 `date '+%Y-%m-%d %H:%M'` 명령으로 조회한다.

```markdown
<!-- generated-by: /plan-doc -->
<!-- generated-at: [YYYY-MM-DD HH:mm] -->
<!-- branch: [브랜치명] -->

# [기능/작업 제목]

## 목표
[이 작업이 해결하는 문제 또는 달성하려는 결과. 1-3문장.]

## 배경
[왜 이 작업이 필요한지. 관련 이슈/티켓 번호, 설계 문서 링크가 있으면 포함.]

## 결정된 구현 방향
[채택된 접근 방식과 그 이유. 검토했으나 기각된 대안이 있으면 간략히 기재.]

## 구현 단계
1. [단계 1]: [설명]
   - 변경 파일: `[파일 경로]`
2. [단계 2]: [설명]
   - 변경 파일: `[파일 경로]`

## 영향 범위
- 영향받는 레이어/모듈: [목록]
- DB/스키마 변경: [있음/없음. 있으면 상세]
- API 변경: [있음/없음. Breaking change 여부]
- 설정 변경: [있음/없음. 있으면 상세]
- 의존성 변경: [있음/없음. 있으면 라이브러리명과 사유]

## 마이그레이션 전략
[DB 스키마 변경 또는 외부 API 계약 변경이 있을 때의 전환 방법. dual-write, feature flag, 무중단 배포 순서 등.]

## 하위 호환성
[기존 기능에 영향을 주는 부분과 대응 방안. 해당 없으면 "해당 없음".]

## 테스트 전략
- Unit: [대상 클래스, Mock 대상]
- Slice/Integration: [대상 범위, 외부 의존성 처리 방식 (Mock/Testcontainers/WireMock)]
- 수동 검증: [필요 시 사유와 검증 절차. 해당 없으면 생략]

## 롤백 시나리오
[배포 후 문제 발생 시 롤백 방법. feature flag, 스키마 호환성 등.]

## 미결 사항
- [ ] [아직 결정되지 않은 항목]
```

### Step 4: 저장 완료 후 출력

저장 완료 후 다음을 출력한다:

```
---
계획 문서가 저장되었습니다: `.claude/plan/{branch-name}-plan.md`

### 문서 요약
- 구현 단계: [N]단계
- 영향 범위: [변경되는 레이어/모듈 목록]
- 미결 사항: [N]건

### 다음 단계
- 구현을 시작하려면 plan 모드를 해제하세요. (Shift+Tab)
- 테스트 케이스를 먼저 도출하려면 `/test`를 실행하세요.
---
```

## 제약

- 현재 대화에서 논의된 내용만 문서화한다. 대화에 없는 내용을 추측하여 채우지 않는다.
- 계획이 수립된 상태란, "무엇을 구현할 것인가"(목표)와 "어떻게 구현할 것인가"(구현 방향 또는 구현 단계) 중 하나 이상이 대화에서 확인 가능한 상태를 말한다. 둘 다 확인할 수 없으면 "현재 대화에서 구현 계획을 찾을 수 없습니다. `/plan`으로 계획을 먼저 수립해 주세요."를 출력하고 중단한다.
- 미결 사항이 있으면 문서 저장은 진행하되, 저장 후 미결 항목을 별도로 안내한다.
- 구현 단계가 10단계를 초과하면, 작업 분할을 권고하는 안내를 저장 후 출력에 추가한다.
