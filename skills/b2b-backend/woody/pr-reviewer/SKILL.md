# PR Reviewer

GitHub PR을 로컬에서 분석하여 코드 리뷰를 수행합니다.
200개 PR 리뷰 분석 결과를 기반으로 한 체크리스트로 리뷰합니다.

## When to Use This Skill

사용 시점:
- "PR 리뷰해줘", "리뷰해줘", "review PR", "/review-pr"
- PR 번호와 함께 사용 (예: "리뷰해줘 4850", "PR 4850 리뷰해줘")
- PR 번호 없이 호출 시 사용자에게 번호를 물어봄

## PR Review Workflow

### Step 1: PR 정보 수집

PR 번호를 받으면 gh CLI로 정보를 수집한다.

```bash
# 메타데이터 조회
gh pr view {PR_NUMBER} --json number,title,body,baseRefName,headRefName,files,additions,deletions,changedFiles,author,isDraft

# diff 조회
gh pr diff {PR_NUMBER}

# 커밋 목록 조회
gh pr view {PR_NUMBER} --json commits --jq '.commits[].messageHeadline'
```

### Step 2: 변경 분석

분석 스크립트를 실행하여 모듈/도메인/파일 분류 정보를 수집한다.

```bash
python .claude/skills/pr-reviewer/scripts/analyze_pr.py --pr {PR_NUMBER}
```

스크립트 출력 (JSON):
- affected_modules: 영향받는 모듈
- detected_domains: 감지된 도메인 (booking, payment, naver 등)
- file_classification: 파일 분류 (source/test/config/migration/docs)
- spring_boot_versions: 모듈별 Spring Boot 버전
- has_test_for_changes: 테스트 존재 여부

### Step 3: 체크리스트 기반 리뷰

Step 1의 diff와 Step 2의 분석 결과를 바탕으로 리뷰를 수행한다.

#### 3-1. 기본 체크리스트 적용
`references/review-checklist.md`를 로드하여 4단계 심각도별 검토:
- **Critical**: DDL 안전성, NPE 위험, 트랜잭션 경계, PII 로깅
- **Major**: 네이밍 컨벤션, 로깅, 아키텍처/레이어, 엣지 케이스, 테스트
- **Suggestion**: 코드 스타일, 인터페이스 분리

#### 3-2. 도메인별 체크리스트 적용
Step 2에서 감지된 도메인에 해당하는 항목만 `references/domain-review-points.md`에서 로드하여 추가 검토.

예: detected_domains가 ["booking", "payment"]이면:
- 예약 도메인: 상태 전이, 예약금 처리, 취소 정책
- 결제 도메인: 결제 상태 검증, 환불 로직, 금액 정합성

#### 3-3. 도메인 전문가 참조 (선택)
필요 시 `.claude/agents/{domain}-expert.md` 파일을 참조하여 도메인 지식 확인.

#### 3-4. Spring Boot 버전 체크
모듈별 Spring Boot 버전을 확인하여:
- 2.7.x 모듈에서 jakarta.* import 사용 시 경고
- 3.3.x 모듈에서 javax.* import 사용 시 경고

### Step 4: 리뷰 결과 터미널 출력

리뷰 결과를 마크다운 형식으로 터미널에 출력한다. `references/review-comment-template.md` 참조.

출력 포맷:
```
## 🤖 [Claude Code Review] PR #{PR_NUMBER}

## 개요
{PR 변경 내용을 1~2문장으로 요약}

## 변경 사항
| 코호트 / 파일 | 요약 |
|---|---|
| **{변경 그룹}** | |
| `{파일명}` | {파일별 변경 요약} |

## 리뷰 요약
| 항목 | 내용 |
|------|------|
| 영향 모듈 | {modules} (Spring Boot {version}) |
| 관련 도메인 | {domains} |
| 변경 규모 | +{additions} / -{deletions} ({files} files) |
| 테스트 포함 | {has_test} |
| 🔴 Critical | {count}건 |
| ⚠️ Major | {count}건 ({요약}) |
| 💡 Suggestion | {count}건 ({요약}) |

### 🎯 예상 리뷰 난이도
{1~5} ({Trivial~Very Complex}) | ⏱️ ~{분}분

## 시
> 🦉 {변경 내용에 맞는 4~6줄 시}

### 🔴 Critical ({count}건)
1. **[카테고리]** `파일경로:라인` - 설명
   - 권장: 수정 방법

### ⚠️ Major ({count}건)
...

### 💡 Suggestions ({count}건)
...

### 🏢 Domain: {도메인명} ({count}건)
...

### 체크리스트 요약
| 카테고리 | 결과 |
|---------|------|
| DDL/마이그레이션 | N/A or N건 |
| NPE 위험 | ... |
| ... | ... |
```

### Step 5: GitHub PR 인라인 리뷰 코멘트 등록

**GitHub Actions 환경** (`claude-code-action`에서 실행 시): 사용자 확인 없이 자동으로 인라인 리뷰 코멘트를 등록한다.
**로컬 환경**: 리뷰 결과 출력 후 사용자에게 확인 ("리뷰 결과를 PR 코멘트로 등록할까요?")

`gh api`로 **인라인 리뷰 코멘트**를 등록한다. 전체 요약은 review body에, 개별 이슈는 파일/라인에 직접 달린다.

```bash
gh api repos/{owner}/{repo}/pulls/{PR_NUMBER}/reviews \
  -X POST \
  --input - <<'EOF'
{
  "event": "COMMENT",
  "body": "## 🤖 [Claude Code Review]\n\n## 개요\n{요약}\n\n## 변경 사항\n{테이블}\n\n## 리뷰 요약\n{테이블}\n\n### 🎯 예상 리뷰 난이도\n{난이도}\n\n## 시\n> 🦉 {시}\n\n---\n<sub>🤖 Generated by [Claude Code](https://claude.ai/claude-code) PR Reviewer</sub>",
  "comments": [
    {
      "path": "{파일경로}",
      "line": {라인번호},
      "body": "🤖 **[Claude Code Review]** {심각도 이모지} **{이슈 제목}**\n\n{상세 설명 + 코드 제안}"
    }
  ]
}
EOF
```

#### 코멘트 작성 규칙

1. **모든 코멘트에 prefix**: `🤖 **[Claude Code Review]**` — 개인 코멘트와 구분
2. **전체 리뷰 body**: 개요 + 변경사항 테이블 + 리뷰 요약 + 난이도 + 시
3. **인라인 코멘트**: 이슈가 있는 정확한 파일/라인에 직접 등록
4. **라인 번호 확인**: `gh api`로 PR 브랜치의 파일 내용을 가져와 정확한 라인 번호를 확인한 후 코멘트 등록
5. **시(poem)**: 변경 내용에 맞는 한국어 시 4~6줄, 🦉 올빼미 테마

#### 라인 번호 확인 방법

```bash
# PR 브랜치의 파일 내용에서 정확한 라인 번호 확인
gh api "repos/{owner}/{repo}/contents/{file_path}" \
  -H "Accept: application/vnd.github.raw" \
  --method GET -F ref={head_branch} | grep -n "{검색어}"
```

**중요**: 코멘트 등록은 반드시 사용자 확인 후 실행. 자동 등록 금지.

## Workflow Example

사용자: "리뷰해줘 4850"

1. `gh pr view 4850 --json ...` → PR 메타데이터
2. `gh pr diff 4850` → 전체 diff
3. `python scripts/analyze_pr.py --pr 4850` → 분석 JSON
4. 분석 결과: modules=[gongbiz-crm-b2b-admin], domains=[sale, ticket]
5. review-checklist.md 로드 → Critical/Major/Suggestion 검토
6. domain-review-points.md에서 sale, ticket 섹션 로드 → 도메인 검토
7. 터미널에 리뷰 결과 출력
8. "리뷰 결과를 PR 코멘트로 등록할까요?" 확인
9. 승인 시 `gh api`로 PR 코멘트 등록

## Troubleshooting

**Issue**: gh CLI 미설치
- **Solution**: `brew install gh` (macOS) 안내

**Issue**: PR 번호가 없는 경우
- **Solution**: 사용자에게 PR 번호 요청

**Issue**: 도메인 감지 실패 (일반적인 경로)
- **Solution**: 기본 체크리스트(Level 1~3)만 적용, 도메인 섹션 생략

**Issue**: diff가 너무 큰 경우
- **Solution**: 파일별로 나누어 리뷰, 핵심 변경 파일 우선 검토
