# 도메인별 PR 리뷰 체크리스트

> 도메인 특성에 따른 리뷰 포인트 정리

이 문서는 `analyze_pr.py`가 감지한 도메인에 해당하는 섹션만 동적으로 적용됩니다.

---

## 예약 (booking)

**복잡도**: 높음
**관련 모듈**: `gongbiz-crm-b2b-backend`, `gongbiz-crm-b2b-api`, `b2c-gongbiz-api`

### 리뷰 포인트
- [ ] 예약 상태 전이 규칙 준수 (BOOKING → SALE → COMPLETE, CANCEL 분기)
- [ ] 예약금(Deposit) 결제/취소 시 상태 동기화
- [ ] 선결제(Prepay) 처리 시 검증 로직
- [ ] 플랫폼별 예약 흐름 고려 (CRM/네이버/ZERO)
- [ ] 예약 시간대 검증 (영업시간 내, 휴게시간 제외)
- [ ] 예약 취소 시 연관 데이터 처리 (결제, 매출, 알림)
- [ ] 동시성 처리 (동일 시간대 중복 예약 방지)

### 주의사항
- 예약 상태 변경 시 반드시 이벤트 발행하여 다른 도메인과 동기화
- 네이버 예약은 `naverBookingId` 필수, 플랫폼 구분 명확히
- ZERO 예약의 경우 초기 상태 결정 로직 확인 (매장결제 on/off, NPay 여부)
- 예약 취소 시 환불 정책 적용 여부 확인

## 결제 (payment)

**복잡도**: 높음
**관련 모듈**: `gongbiz-crm-b2b-backend`, `gongbiz-crm-b2b-payment-domain`, `gongbiz-crm-b2b-payment-infrastructure`

### 리뷰 포인트
- [ ] 결제 상태 검증 (PENDING → PAID → CANCELLED/REFUNDED)
- [ ] 부분 취소/전액 취소 로직 구분
- [ ] 환불 금액 계산 정합성 (`totalAmount - cancelledAmount`)
- [ ] PG사별 연동 로직 (KIS, NicePay) 차이 고려
- [ ] 쿠폰/할인 적용 후 취소 시 복원 처리
- [ ] 결제 실패 시 롤백 처리
- [ ] 예약-결제 연결 (`BookingPayment`) 무결성

### 주의사항
- 결제 취소 시 반드시 PG사 API 호출 결과 확인 후 상태 변경
- 쿠폰 사용 후 취소 시 쿠폰 상태 복원 (`CouponState.READY`)
- 금액 계산 시 소수점 처리 주의 (`BigDecimal` 사용)
- 네이버페이 결제는 `isUsedNaverPay = true` 설정 필수

## 정산 (settlement)

**복잡도**: 높음
**관련 모듈**: `gongbiz-crm-settlement-batch`, `gongbiz-crm-b2b-payment-domain`, `gongbiz-crm-b2b-payment-infrastructure`

### 리뷰 포인트
- [ ] 정산 대상 조회 기준 명확성 (매핑 생성일자 기준)
- [ ] 수수료 계산 로직 정합성 (매출 등급별 수수료율 적용)
- [ ] 차액정산 처리 (영세/중소 가맹점)
- [ ] 중복 요청 방지 (이미 처리된 정산 건 제외)
- [ ] 배치 실행 로그 및 예외 처리
- [ ] PG사별 정산 파라미터 검증 (`acntSndNm` 등)

### 주의사항
- 수수료율 변경 시 기존 정산 건에 영향 없도록 시점 기준 명확히
- 국민카드 반송 등 카드사별 예외 케이스 처리
- 정산 상태 전이 시 이력 기록 (`SettlementRequestStatus`)
- ShedLock으로 배치 중복 실행 방지 확인

## 고객 (customer)

**복잡도**: 낮음
**관련 모듈**: `gongbiz-crm-b2b-backend`, `gongbiz-crm-b2b-api`, `b2c-gongbiz-api`

### 리뷰 포인트
- [ ] 고객 개인정보(PII) 마스킹 처리
- [ ] 중복 고객 처리 (전화번호 기준 통합)
- [ ] 고객 삭제 시 연관 데이터 영향도 (예약, 매출)

- 개인정보 조회 시 권한 검증 필수
- GDPR 대응: 고객 탈퇴 시 개인정보 삭제/익명화

## 시술 (cosmeticproc)

**복잡도**: 낮음
**관련 모듈**: `gongbiz-crm-b2b-backend`, `gongbiz-crm-b2b-api`

### 리뷰 포인트
- [ ] 시술 메뉴 가격 변경 시 기존 예약에 영향 없도록
- [ ] 시술 소요 시간 변경 시 예약 스케줄 재검증
- [ ] 시술 삭제 시 연관 예약 확인

- 시술 소요 시간은 네이버 상품 정보와 동기화 필요
- 시술 비활성화 시 네이버 상품도 함께 비활성화

**관련 모듈**: `gongbiz-crm-b2b-backend`, `gongbiz-crm-b2b-api`

### 리뷰 포인트

## 알림 (notification)

**복잡도**: 중간
**관련 모듈**: `gongbiz-notification`, `gongbiz-notification-orchestrator`, `gongbiz-notification-batch`

### 리뷰 포인트
- [ ] 알림톡 템플릿 매칭 (템플릿 코드와 변수 일치)
- [ ] 발송 실패 시 재시도 로직 또는 로깅
- [ ] 비동기 처리 확인 (Kafka, SQS)
- [ ] 알림 발송 이력 기록
- [ ] 알림 수신 거부 고객 필터링

### 주의사항
- 알림톡 템플릿은 카카오 승인 후 사용 가능
- 발송 실패 시 사용자에게 알림 없이 로깅만 수행
- 예약 확정/취소 알림은 즉시 발송, 리마인드는 배치 처리
- 알림 발송 시 개인정보 마스킹 확인

## 네이버 (naver)

**복잡도**: 높음
**관련 모듈**: `gongbiz-crm-b2b-backend`, `gongbiz-crm-b2b-infrastructure`, `gongbiz-crm-b2b-batch`

### 리뷰 포인트
- [ ] 네이버 API 호출 실패 시 로깅 (에러 코드, 메시지)
- [ ] 네이버 결제 상태 검증 (POST+REGISTERED 상태만 저장)
- [ ] 네이버 웹훅 중복 처리 방지 (idempotency key)
- [ ] 네이버 스케줄 동기화 실패 시 재시도 로직
- [ ] 네이버 예약 상태 매핑 정합성 (네이버 BOOKED → CRM BOOKING)
- [ ] NPay 활성 샵에서 예약 확정 방식 정책 준수

### 주의사항
- 네이버 API 호출은 배치 또는 이벤트 기반 비동기 처리
- 네이버 예약 생성 시 `naverBookingId` 필수 저장
- NPay 매장결제 상태는 네이버 웹훅으로 수신 (POST → PAID)
- 네이버 스케줄 동기화는 직원 영업시간 + 휴게시간 기반
- 네이버 연동 실패 시 Slack 알림 발송

**복잡도**: 중간
**관련 모듈**: `gongbiz-crm-b2b-backend`, `gongbiz-crm-b2b-api`

### 리뷰 포인트

- 정기권은 만료 전 자동 갱신 알림 발송

## 영업일 (businessday)

**복잡도**: 중간
**관련 모듈**: `gongbiz-crm-b2b-backend`, `gongbiz-crm-b2b-infrastructure`

### 리뷰 포인트
- [ ] 영업 시작 시간 < 종료 시간 검증
- [ ] 24:00 케이스 처리 (자정 넘김)
- [ ] 요일별 영업시간 범위 중복 검증
- [ ] 휴게시간이 영업시간 범위 내인지 검증
- [ ] 휴게시간 중복 검증 (시간대 겹침 없음)
- [ ] 직원 영업시간이 샵 영업시간 범위 내인지 검증

### 주의사항
- 영업 종료 시간 24:00은 익일 00:00으로 처리
- 휴게시간은 여러 개 설정 가능하지만 겹치면 안 됨
- 직원 영업시간은 샵 영업시간을 벗어날 수 없음
- 네이버 스케줄 동기화 시 영업시간 기준으로 판매 가능 시간 계산
- 공휴일/휴무일 설정 시 예약 불가 처리

## 리뷰 자동화 가이드

### 도메인 감지 로직 (analyze_pr.py)

```python
# 변경된 파일 경로를 기반으로 도메인 자동 감지
DOMAIN_PATTERNS = {
    'booking': ['booking', 'book'],
    'payment': ['payment', 'pay'],
    'settlement': ['settlement', 'commission'],
    'sale': ['sale'],
    'shop': ['shop'],
    'customer': ['customer', 'client'],
    'cosmeticproc': ['cosmetic', 'procedure', 'menu'],
    'employee': ['employee', 'staff'],
    'notification': ['notification', 'alimtalk', 'sms'],
    'naver': ['naver', 'npay'],
    'ticket': ['ticket', 'subscription'],
    'businessday': ['businessday', 'worktime', 'breaktime']
}
```

### 리뷰 체크리스트 적용

1. PR 분석 스크립트가 변경된 파일 경로에서 도메인 키워드 감지
2. 감지된 도메인에 해당하는 섹션만 리뷰 코멘트에 포함
3. 복잡도가 '높음'인 도메인은 반드시 해당 도메인 전문가 리뷰 권장

### 복잡도별 리뷰 전략

| 복잡도 | 도메인 | 리뷰 전략 |
|--------|--------|-----------|
| **높음** | 예약, 결제, 정산, 네이버 | 도메인 전문가 리뷰 필수, 단위 테스트 필수 |
| **중간** | 매출, 샵, 직원, 알림, 이용권, 영업일 | 체크리스트 기반 셀프 리뷰 가능, 통합 테스트 권장 |
| **낮음** | 고객, 시술 | 기본 검증만으로 충분, 단위 테스트 권장 |
