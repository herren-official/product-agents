# 외부 리서치 (Research)

## 워크플로우

### Step 1: 조사 대상 확인

루나의 산출물에서 "외부 연동 필요 항목"을 읽고 조사 대상을 확정한다.

### Step 2: 기존 연동 코드 탐색

프로젝트 내 해당 외부 시스템 연동 코드가 있는지 먼저 확인한다.

**탐색 패턴:**
- 패키지명/클래스명에 외부 시스템명 포함 (예: `naver`, `shopby`, `nicepay`)
- `infrastructure` 레이어의 Client/Adapter 클래스
- application.yml의 외부 URL/키 설정
- 기존 API 호출 패턴 (RestTemplate, WebClient, Feign 등)

### Step 3: 공식 문서 조사

WebSearch/WebFetch로 공식 문서를 조사한다. context7 MCP가 있으면 우선 사용한다.

**조사 항목:**
- API 엔드포인트 + 요청/응답 스펙
- 인증/인가 방식
- rate limit, 데이터 크기 제한
- 버전 요구사항, 지원 종료 일정
- 에러 코드 및 재시도 정책
- **이미지/파일 업로드 흐름 (v2 신설)** — 사전 업로드 vs URL 전달 vs base64 인라인 — 어느 방식?
- **TTL/만료 정책** (fileKey, imgUrl, 토큰)
- **동일 자원 재요청 멱등성** (같은 파일 재업로드 시 같은 키 반환?)

### Step 3.5: 벤더명·서비스명 정확성 교차 확인 (v2 신설 — 2026-04-26)

> 도입 배경: 2026-04-26 LG CNS = 비즈뿌리오로 잘못 가정한 사례.
> 실제로는 비즈뿌리오(인포뱅크 자회사) ≠ LG CNS 톡드림(LG CNS 자체).
> 잘못된 가정으로 후속 설계 전체가 흔들렸음.

**교차 확인 절차**:
1. 기획서/요구사항이 언급한 벤더명을 추출 ("LG CNS", "인포뱅크" 등)
2. **코드베이스 endpoint grep으로 실제 사용 서비스 확인**:
   ```
   grep -rn "https?://" {모듈}/src/main/resources/application*.{yml,properties}
   grep -rn "https?://[^/]*" {모듈}/src/main/**/*.{kt,java}
   ```
3. endpoint 도메인으로 정확한 서비스명 식별:
   - `talkapi.lgcns.com` → LG CNS 톡드림 (TalkAPI)
   - `omni.ibapi.kr` → 인포뱅크 OMNI
   - `api.bizppurio.com` → 비즈뿌리오 (인포뱅크 자회사)
4. **벤더명·서비스명·운영사를 명확히 산출물에 명시**
5. 동명·유사 서비스 혼동 가능성 경고

### Step 3.6: 공개 API 부재 가능성 검출 (v2 신설)

> 모든 외부 API가 공개 문서를 가진다고 가정 금지.
> 폐쇄형 B2B 서비스는 영업 미팅 없이 결정 불가.

**판정 기준**:
- 공개 개발자 포털·Swagger·GitBook 부재 → "폐쇄형" 플래그
- 영업 페이지(예: `bizmsg.lgcns.com`)만 노출 → 폐쇄형 가능성 높음
- API key 발급 절차가 영업 컨택 의무 → 폐쇄형 확정

**폐쇄형 판정 시 산출물 처리**:
- 산출물에 "폐쇄형 — 영업 미팅 필요" 명확히 표기
- 영업 컨택 포인트 명시 (이메일/전화/계약 담당자)
- 코드베이스 endpoint·인증 패턴으로 추정 가능한 부분만 기록
- 미확인 항목은 모두 "영업 확인 필요"로 분리

#### 추가 필수 섹션 (v2 — 2026-04-26)

##### 외부 의존 운영 전제 매트릭스 (필수)

설계서 9.X와 1:1 매핑되는 매트릭스를 산출물에 반드시 포함:

```markdown
| 외부 시스템 | 정확명 (운영사·서비스) | 공개 문서 | dev 계정 | prod 계정 | API key | 테스트 환경 | 영업 컨택 | 미확보 항목 |
|------------|---------------------|:---:|:---:|:---:|:---:|------|---------|-----------|
| 인포뱅크 OMNI | (주)인포뱅크 OMNI API v2 | ✅ GitBook | ❓ | ❓ | ❓ | ❓ | mpush@... | 계정 |
| LG CNS 톡드림 | LG CNS TalkAPI | ❌ 폐쇄 | ✅ | ✅ | ✅ | devtalkapi | 기존 | - |
```

❓ = 미확보. 모든 ❓는 R-XX 등록 필요 (Decision Gate 표준 적용).
