프로젝트 경로: `~/herren-repo/projects/{프로젝트명}/`

## 모드 분기

`$ARGUMENTS`가 `update`이면 → **업데이트 모드** 실행. 그 외 → **조회 모드** 실행.

## 조회 모드 (기본)

1. **프로젝트 선택**: `$ARGUMENTS`로 폴더 매칭. 없으면 목록 보여주고 선택 요청.
2. **CONTEXT.md 읽기**: 상태, 테이블, 코드 진입점, PR, 세션 기록 파악.
3. **하위 파일 스캔**: 트리 출력. `.md`/`.sql`은 헤더 읽기, `.csv`는 행 수만 확인.
4. **요약 출력 후 지시 대기**.

## 업데이트 모드 (`/pc update`)

현재 대화에서 진행된 작업을 기반으로 CONTEXT.md를 업데이트한다.

1. **프로젝트 특정**: 현재 대화에서 작업 중인 프로젝트 폴더 특정. 모호하면 질문.
2. **CONTEXT.md 읽기**: 현재 상태 확인.
3. **업데이트 항목 판단** (해당되는 것만):
   - `현재 상태`: 단계, 마지막 세션 날짜
   - `다음 할 일`: 완료된 항목 제거, 새 항목 추가
   - `핵심 설계 결정`: 이번 세션에서 내린 결정 추가
   - `발견한 버그`: 새로 발견한 버그 추가
   - `PR 현황`: PR 생성/상태 변경 반영
   - `세션 기록`: 새 세션 블록 추가 (목표, 수행 작업, 결과, 이슈/배운 점)
   - `참조`: 새 파일/경로 추가
4. **변경사항 적용 후 요약 출력**.
