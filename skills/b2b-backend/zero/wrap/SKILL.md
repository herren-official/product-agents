현재 세션을 분석하여 마무리 작업을 수행한다. 5개의 에이전트를 활용하여 세션에서 발생한 학습, 패턴, 자동화 기회를 체계적으로 정리한다.

## Phase 1: 병렬 분석 (4개 에이전트 동시 실행)

Task 도구를 사용하여 아래 4개 에이전트를 **동시에** 실행한다. 각 에이전트는 현재 대화 컨텍스트에 접근 가능하므로, 세션 내용을 직접 분석할 수 있다.

### Agent 1: doc-updater (CLAUDE.md 업데이트 제안)

```
subagent_type: Explore
name: doc-updater
```

이 세션에서 발견된 내용을 바탕으로 CLAUDE.md 파일 업데이트를 제안한다.

작업 내용:
1. 현재 대화 컨텍스트를 분석하여 다음을 파악:
   - 새로 발견된 프로젝트 패턴이나 컨벤션
   - 자주 사용된 명령어나 워크플로우
   - 프로젝트 구조에 대한 새로운 이해
   - 작업 선호사항이나 규칙
2. `~/herren-repo/CLAUDE.md` (개인 규칙)를 읽고 기존 내용과 비교
3. 프로젝트 CLAUDE.md가 있다면 해당 파일도 읽고 비교
4. 중복되지 않는 새로운 내용만 추가 제안으로 정리
5. 결과를 다음 형식으로 반환:
   - **추가 제안**: 새로 추가할 내용 목록
   - **수정 제안**: 기존 내용 중 업데이트가 필요한 항목
   - **대상 파일**: 어느 CLAUDE.md에 반영할지 명시

### Agent 2: learning-extractor (학습 추출)

```
subagent_type: Explore
name: learning-extractor
```

이 세션에서 얻은 학습과 인사이트를 추출한다.

작업 내용:
1. 현재 대화 컨텍스트를 분석하여 다음을 추출:
   - 해결한 문제와 그 해결 방법
   - 발견한 코드 패턴이나 아키텍처 인사이트
   - 디버깅 과정에서 얻은 교훈
   - 외부 서비스/API에 대한 새로운 이해
   - 도메인 지식 (비즈니스 로직, 데이터 흐름 등)
2. `$HOME/.claude/projects/<user-home-encoded>/memory/MEMORY.md`를 읽고 기존 메모리와 비교 (`<user-home-encoded>`는 사용자 홈 디렉토리에서 `/`를 `-`로 치환한 형태. 예: `/Users/herren` → `-Users-herren`)
3. 메모리 디렉토리의 다른 파일도 확인하여 중복 방지
4. 결과를 다음 형식으로 반환:
   - **새로운 학습**: 기존 메모리에 없는 새로운 인사이트
   - **업데이트 필요**: 기존 메모리 중 수정/보강이 필요한 항목
   - **저장 제안**: 어느 메모리 파일에 저장할지 (MEMORY.md 또는 별도 토픽 파일)

### Agent 3: followup-suggester (후속 작업 제안)

```
subagent_type: Explore
name: followup-suggester
```

이 세션을 바탕으로 다음에 할 수 있는 후속 작업을 제안한다.

작업 내용:
1. 현재 대화 컨텍스트를 분석하여 다음을 파악:
   - 세션에서 완료하지 못한 작업
   - 세션 중 발견했지만 다루지 않은 이슈
   - 현재 작업의 자연스러운 다음 단계
   - 관련된 개선 기회 (리팩토링, 테스트 추가 등)
   - 탐색이 더 필요한 영역
2. 각 제안에 우선순위(높음/중간/낮음)와 간단한 설명 포함
3. 결과를 다음 형식으로 반환:
   - **미완료 작업**: 이번 세션에서 끝내지 못한 것
   - **발견된 이슈**: 세션 중 발견한 문제점
   - **개선 기회**: 코드/프로세스 개선 제안
   - **탐색 필요**: 더 알아봐야 할 영역

### Agent 4: automation-scout (자동화 기회 탐지)

```
subagent_type: Explore
name: automation-scout
```

이 세션에서 반복적으로 수행한 작업 중 자동화할 수 있는 것을 탐지한다.

작업 내용:
1. 현재 대화 컨텍스트를 분석하여 다음을 파악:
   - 반복적으로 실행한 명령어나 패턴
   - 매번 수동으로 수행하는 체크/검증 작업
   - 스킬이나 훅으로 만들면 좋을 워크플로우
   - git hook, pre-commit, CI/CD로 자동화 가능한 작업
2. `~/.claude/skills/` 디렉토리를 확인하여 이미 존재하는 스킬과 비교
3. 결과를 다음 형식으로 반환:
   - **스킬 후보**: 새 Claude Code 스킬로 만들면 좋을 것
   - **훅 후보**: git hook이나 Claude Code hook으로 자동화할 것
   - **워크플로우 개선**: 기존 작업 흐름을 더 효율적으로 만들 수 있는 방법

## Phase 2: 중복 검증

Phase 1의 4개 에이전트 결과를 모두 받은 후, 5번째 에이전트를 실행한다.

### Agent 5: duplicate-checker (중복 검증)

```
subagent_type: Explore
name: duplicate-checker
```

Phase 1 에이전트들의 결과를 취합하고 중복을 제거한다.

작업 내용:
1. 4개 에이전트의 결과를 받아 중복 항목 식별
2. 기존 CLAUDE.md, 메모리 파일과의 중복 최종 확인
3. 최종 정리된 결과를 반환

## Phase 3: 사용자에게 결과 제시

모든 에이전트 결과를 정리하여 사용자에게 다음 형식으로 보여준다:

```
## 세션 요약

### 학습 & 인사이트
- (learning-extractor 결과)

### CLAUDE.md 업데이트 제안
- (doc-updater 결과)

### 후속 작업
- (followup-suggester 결과)

### 자동화 기회
- (automation-scout 결과)
```

그 후 사용자에게 다음 액션을 선택하도록 한다:
1. **메모리 저장**: 학습 내용을 메모리 파일에 저장
2. **CLAUDE.md 업데이트**: 제안된 내용을 CLAUDE.md에 반영
3. **자동화 생성**: 제안된 스킬/훅을 실제로 생성
4. **전체 적용**: 위 모든 항목 적용
5. **건너뛰기**: 아무것도 하지 않음

## 인자 처리

$ARGUMENTS가 제공된 경우 해당 내용을 추가 컨텍스트로 활용하여 분석한다.
예: `/wrap 오늘 SMS 관련 코드를 많이 봤어` -> SMS 관련 학습에 집중하여 분석
