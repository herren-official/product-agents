# 비즈니스 도메인 가이드

이 skill은 프로젝트의 비즈니스 도메인을 이해하기 위한 배경 지식을 제공합니다.

## 서비스 개요

공비즈 CRM B2B 웹은 **미용실/뷰티샵을 위한 고객 관리 및 예약 시스템**입니다.

### 주요 사용자

- **샵 오너/관리자**: 매장 운영 관리, 직원 관리, 매출 분석
- **직원**: 예약 관리, 고객 시술, 매출 기록

## 주요 엔티티

### 1. 고객 (Customer)

- 샵을 방문하는 손님
- 고객 정보, 방문 이력, 시술 이력 관리
- 포인트 관리

### 2. 예약 (Booking/Reservation)

- 고객의 방문 예약
- 담당 직원, 시술 항목, 예약 시간 포함
- 상태: 예약대기, 확정, 완료, 취소, 노쇼

### 3. 직원 (Employee)

- 샵에서 근무하는 스태프
- 담당 시술, 근무 시간, 인센티브 설정
- 직급/역할 구분 (원장, 디자이너, 어시스턴트 등)

### 4. 시술/메뉴 (Cosmetic/Menu/SaleItem)

- 샵에서 제공하는 서비스 항목
- 시술 종류: 커트, 펌, 염색, 클리닉 등
- 가격, 소요 시간, 담당 직원 정보

### 5. 매출 (Sales/Payment)

- 시술 후 결제 정보
- 결제 수단: 현금, 카드, 계좌이체
- 할인, 멤버십 적용, 포인트 사용

### 6. 샵 (Shop)

- 매장 정보
- 영업 시간, 휴무일, 연락처
- 샵 설정 (예약 정책, 알림 설정 등)

## 주요 화면 구조

### 캘린더/예약 관리

- `/calendar`: 예약 캘린더 뷰
- 일별/주별/월별 예약 현황 확인
- 드래그 앤 드롭으로 예약 시간 변경

### 고객 관리

- `/customer`: 고객 목록
- `/customer/:id`: 고객 상세 (방문 이력, 시술 이력)
- 고객 검색, 필터링

### 매출/통계

- `/statistics`: 통계 대시보드
- 일별/월별 매출 분석
- 직원별, 시술별 매출 통계

### 설정

- `/settings`: 샵 설정
- 직원 관리, 인센티브 설정
- 시술 메뉴 관리, 가격 설정

## 주요 용어

| 용어 | 설명 |
|------|------|
| **노쇼 (No-Show)** | 예약 후 연락 없이 미방문 |
| **인센티브** | 직원 성과 기반 추가 급여 |
| **예약금** | 예약 시 미리 받는 선결제 금액 |
| **포인트** | 결제 시 적립/사용 가능한 포인트 |
| **시술** | 미용 서비스 (커트, 펌, 염색 등) |
| **담당자** | 시술을 진행하는 직원 |
| **매출 차감** | 환불, 할인 등으로 인한 매출 감소 |

## API 응답 타입 네이밍

- `*ResponseDto`: API 응답 데이터
- `*RequestDto`: API 요청 데이터
- `*Detail`: 상세 정보
- `*Summary`: 요약 정보
- `*List`: 목록 데이터

## 도메인 폴더 구조

```
src/domains/
├── calendar/          # 캘린더/예약
├── customer/          # 고객 관리
├── statistics/        # 매출/통계
├── employee/          # 직원 관리
├── settings/          # 설정
└── shop/              # 샵 정보
```
