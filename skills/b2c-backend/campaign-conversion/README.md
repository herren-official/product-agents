# campaign-conversion 스킬

알림톡/마케팅 메시지 발송 후 **예약 전환, 회원가입 전환, 쿠폰 사용 여부**를 분석하는 스킬입니다.

## 설치 방법

### 1. 스킬 폴더 복사

`campaign-conversion` 폴더를 OpenClaw 워크스페이스의 `skills/` 디렉토리에 복사합니다.

```bash
cp -r campaign-conversion ~/.openclaw/workspace/skills/
```

### 2. 환경변수 설정 (관리자 1회만)

OpenClaw 환경에 아래 3개의 환경변수가 등록되어 있어야 합니다.
이미 등록되어 있다면 이 단계는 건너뜁니다.

```
GONGBIZ_DB_HOST
GONGBIZ_DB_USER
GONGBIZ_DB_PASSWORD
```

스킬이 로드될 때 자동으로 환경변수를 가져오므로, 마케터가 별도로 설정할 필요는 없습니다.

### 3. 스킬 로드 확인

```bash
openclaw skills list
```

`campaign-conversion`이 목록에 나타나면 설치 완료입니다.

## 사용 방법

### 방법 1: 노션 링크로 요청

노션에 작성된 데이터 추출 요청 페이지 링크를 전달합니다.

```
이 요청 전환 데이터 분석해줘
https://notion.so/0909/잔여-쿠폰-알림톡-...
```

스킬이 노션 페이지에서 대상 파일, 제외 파일, 분석 기간 등을 자동으로 파싱합니다.
분석 기간이 불명확하면 추가 질문을 합니다.

### 방법 2: 직접 요청

노션 링크 없이도 사용할 수 있습니다.

```
쿠폰 알림톡 전환 데이터 분석해줘
```

스킬이 순차적으로 필요한 정보를 질문합니다:
1. 대상자 파일
2. 제외할 파일 (없으면 없다고 답변)
3. 분석 시작일시 (발송 시각)
4. 분석 종료일시

---

## 실행 과정

```
요구사항 수집 → 요약 및 확인 → 제외 대상 중복 제거 → DB 전환 데이터 조회 → 결과 엑셀 출력 → 채널에 업로드
```

1. 요구사항을 수집하고 요약을 보여준 뒤 확인을 받습니다.
2. 제외 파일이 있으면 전화번호 기준으로 중복을 제거합니다.
3. 정제된 전화번호로 DB에서 예약/가입/쿠폰 데이터를 조회합니다.
4. 결과 엑셀 파일을 생성하고 채널에 업로드합니다.
5. 분석 결과 요약 코멘트를 함께 남깁니다.

## 출력 결과

### 결과 파일

- 형식: 엑셀(.xlsx), 모든 컬럼 텍스트 타입
- 파일명: `{날짜}_{캠페인유형}_예약전환_회원가입_쿠폰사용여부.xlsx`

### 분석 컬럼

| 컬럼 | 값 | 설명 |
|---|---|---|
| 예약여부 | 예약 / 미예약 | 분석 기간 내 유효 예약 여부 |
| 회원가입여부 | 신규가입 / 기존가입 / 미가입 | 분석 기간 기준 가입 상태 |
| 쿠폰사용여부 | O / X / (빈값) | 해당 예약의 쿠폰 사용 여부 |

---

## 주의사항

- **분석 시작일시**는 날짜뿐 아니라 발송 시각(시:분:초)까지 필요합니다.
- **분석 종료일시**는 날짜만 입력하면 자동으로 23:59:59가 적용됩니다.
- 전화번호 앞의 0이 빠져 있는 엑셀 파일도 자동으로 보정됩니다.
- DB는 읽기 전용 Replica를 사용하므로 운영 DB에 영향을 주지 않습니다.
