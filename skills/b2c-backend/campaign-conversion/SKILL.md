# 캠페인 전환 데이터 분석

마케팅 메시지(알림톡, 리텐션 메시지 등) 발송 후 고객의 예약 전환, 회원가입 전환, 쿠폰 사용 여부를 분석한다.

## 전체 플로우

```
요구사항 수집 → 요약 및 확인 → 제외 대상 중복 제거 → DB 전환 데이터 조회 → 결과 출력
```

---

## STEP 1: 요구사항 수집

두 가지 모드를 지원한다. 사용자가 노션 링크를 제공하면 모드 A, 아니면 모드 B로 진행한다.

### 모드 A: 노션 링크 제공 시

1. Notion MCP로 페이지를 읽는다.
2. 페이지에서 아래 정보를 파싱한다:
   - **캠페인 유형**: 제목에서 추출 (리텐션, 회원권, 쿠폰, 이벤트 등)
   - **분석 시작일시**: "조건" 섹션에서 발송 시점 파악. 날짜뿐 아니라 **시간**까지 필요하다.
   - **분석 종료일시**: "조건" 섹션에서 "X일까지" 형태로 추출. 날짜만 있으면 `23:59:59`를 붙인다.
   - **대상 파일**: 발송 대상자 엑셀 파일 (첨부파일 중 "발송 인원 전체 파일" 등)
   - **제외 파일**: "제외 필요"로 표시된 첨부 엑셀 파일들
3. 분석 기간이 명확하지 않으면 사용자에게 질문한다.

### 모드 B: 노션 링크 없이 시작 시

사용자에게 순차적으로 질문한다:

1. "대상자 파일(엑셀)을 알려주세요."
2. "제외할 파일이 있나요? 있다면 알려주세요. (여러 개 가능)"
3. "분석 시작일시를 알려주세요. (발송 시각, 예: 2026-04-02 14:00:00)"
4. "분석 종료일시를 알려주세요. (예: 2026-04-09, 기본값: 발송 후 7일)"
   - 날짜만 입력하면 `23:59:59`를 붙인다.

### 공통: 요약 출력 및 확인

어떤 모드든 실행 전에 반드시 아래 형태로 요약하고 사용자 확인을 받는다:

```
[요구사항 요약]
- 캠페인: 잔여 쿠폰 알림톡
- 대상 파일: 20260402_쿠폰_보유고객.xlsx (시트 1개)
- 제외 파일: 없음
- 분석 시작일시: 2026-04-02 14:00:00
- 분석 종료일시: 2026-04-09 23:59:59

[추출 데이터]
- 예약 전환 여부 (예약 / 미예약)
- 회원가입 전환 여부 (신규가입 / 기존가입 / 미가입)
- 쿠폰 사용 여부 (O / X)
- 예약 상세 (platform, bookno, shopno, bookdate, booktime, content, state, regdate)
- 회원 상세 (가입일, 회원상태)

추가로 필요한 조건이 있으면 말씀해주세요. 없으면 "진행"이라고 답해주세요.
```

## STEP 2: 제외 대상 중복 제거

제외 파일이 있는 경우에만 수행한다. 없으면 이 단계를 건너뛴다.

### 2-1. 전화번호 컬럼 자동 감지

대상 파일과 제외 파일 모두에서 전화번호 컬럼을 자동으로 찾는다.

**감지 우선순위 (컬럼명 키워드):**

1. `customer_contact` (정확 매칭)
2. `대상자전화번호` (정확 매칭)
3. `contact` 포함 (대소문자 무시)
4. `연락처` 포함
5. `전화번호` 포함
6. `휴대폰` 포함
7. `phone` 포함 (대소문자 무시)
8. `cell` 포함 (대소문자 무시)
9. `mobile` 포함 (대소문자 무시)

**키워드 매칭 후보가 여러 개일 경우:**

각 후보 컬럼의 데이터를 샘플링하여 한국 전화번호 패턴 매칭 비율을 계산한다:
```
01[0-9]{8,9}      → 010, 011 등 (11자리)
1[0-9]{9}          → 0이 빠진 경우 (10자리)
8210[0-9]{8}       → 국제번호 형식
```

- 후보 1개 + 패턴 확인 → 자동 채택
- 후보 여러 개 + 패턴으로 1개만 높음 → 자동 채택 (가장 높은 것)
- 후보 여러 개 + 패턴도 비슷 → 사용자에게 질문
- 후보 0개 + 패턴으로 발견 → 자동 채택
- 후보 0개 + 패턴도 없음 → 사용자에게 질문

### 2-2. 전화번호 정규화

대상 파일과 제외 파일 양쪽 모두에 적용한다. 엑셀에서 전화번호 앞의 0이 빠지는 문제가 빈번하다.

```
정규화 규칙:
- 10자리 숫자 (0으로 시작하지 않음) → 앞에 0 추가 (예: 1012345678 → 01012345678)
- 9자리 숫자 (1로 시작) → 앞에 0 추가 (예: 101234567 → 0101234567)
- +82 또는 82 접두사 → 0으로 변환
- 하이픈, 공백 제거
- 그 외 → 그대로 유지
```

### 2-3. 매칭 및 제거

1. 제외 파일(들)의 모든 시트에서 전화번호를 추출하고 정규화하여 **하나의 제외 집합**을 만든다.
2. 각 대상 파일을 **독립적으로** 처리한다:
   - 대상 파일의 각 시트에서 전화번호를 정규화한다.
   - 대상 파일의 전화번호가 제외 집합에 포함되면 해당 행을 제거한다.
3. 제거 결과를 대상 파일별로 보고한다:
   ```
   [중복 제거 결과]
   - 쿠폰_사용가능_고객.xlsx: 15,000건 → 제외 634건 → 정제 후 14,366건
   - 쿠폰_사용불가_고객.xlsx: 15,000건 → 제외 600건 → 정제 후 14,400건
   ```

### 파일 처리 규칙

- **대상자 파일은 독립적으로 처리**한다. 대상자 파일 N개 → 결과 파일 N개
- **제외 파일은 모든 대상자 파일에 공통 적용**한다.
- 대상자 파일 하나에 시트가 여러 개여도 결과는 하나의 CSV로 통합한다.
- 분석 기간이 하나만 제공되면 모든 시트에 공통 적용한다.
- 시트명이 `YYMMDD` 형식이고 시트별로 발송일이 다른 경우, 시트명에서 날짜를 추출하여 시트별로 다른 분석 시작일을 적용할 수 있다. (YY < 50이면 2000년대)
  - 시트 내에 `발송시간` 컬럼이 있으면 해당 시간을 분석 시작 시각으로 사용한다.
  - 판단이 어려우면 사용자에게 질문한다.

## STEP 3: DB 전환 데이터 조회

정제된 전화번호 목록으로 DB에서 전환 데이터를 조회한다.
배치 처리(100명씩)로 DB 부하를 최소화한다.

### 3-1. DB 접속 정보

환경변수에서 읽어온다:

```
Host:     $GONGBIZ_DB_HOST
User:     $GONGBIZ_DB_USER
Password: $GONGBIZ_DB_PASSWORD
Database: nailshop
Charset:  utf8mb4
```

**주의: 이 DB는 Replica(읽기 전용)이다. SELECT만 실행할 것.**

### 3-2. 회원가입 전환 조회

```sql
SELECT cell_phone, created_datetime, status
FROM `gongbiz-b2c`.zero_user
WHERE cell_phone IN ({전화번호_목록})
```

**가입 상태 판단:**
- `신규가입`: `created_datetime`이 분석 시작일시 ~ 분석 종료일시 사이
- `기존가입`: `created_datetime`이 분석 시작일시 이전
- `미가입`: 조회 결과 없음, 또는 status가 탈퇴 상태(`WITHDRAW`, `inactive` 등), 또는 `created_datetime`이 분석 종료일시 이후

### 3-3. 예약 전환 조회

bookno는 `YYYYMMDD` + 시퀀스로 구성된 PK다. 인덱스를 활용하기 위해 bookno 범위 조건을 추가한다.

```sql
SELECT contact, platform, bookno, shopno, bookdate, booktime,
       content, state, regdate
FROM nailshop.booking
WHERE bookno >= '{시작일_YYYYMMDD}0000000000'
  AND contact IN ({전화번호_목록})
  AND regdate >= '{분석_시작일시}'
  AND regdate <= '{분석_종료일시}'
  AND state IN ('예약', '방문완료', '시술중', '매출')
```

**예약 상태 판단:**
- `예약`: 위 쿼리 결과가 있으면 예약 전환. 한 고객에 여러 예약이 있을 수 있으며, 각 예약별로 행을 생성한다.
- `미예약`: 위 쿼리 결과가 없는 경우

### 3-4. 쿠폰 사용 여부 조회

예약이 있는 경우에만 실행한다. 예약번호(bookno)로 쿠폰 사용 정보를 조회한다.

```sql
SELECT bookno, status
FROM `gongbiz-b2c`.zero_user_coupon
WHERE bookno IN ({예약번호_목록})
```

**쿠폰 사용 판단:**
- `O`: 해당 bookno에 `status='USED'`인 쿠폰이 있는 경우
- `X`: 예약은 있으나 USED 상태의 쿠폰이 없는 경우
- 빈값: 예약이 없는 경우

## STEP 4: 결과 출력

### 출력 형식

엑셀(.xlsx) 파일로 출력한다. **모든 컬럼은 텍스트(문자열) 타입으로 저장**해야 한다. 숫자 타입으로 저장하면 전화번호 앞의 0이 잘린다.

### 컬럼 순서

원본 엑셀의 모든 컬럼을 유지하고, 뒤에 분석 결과 컬럼을 추가한다:

```
[원본 컬럼들] + 예약여부 | platform | bookno | 쿠폰사용여부 | booking_shopno | bookdate | booktime | content | state | regdate | 가입일 | 회원상태 | 회원가입여부
```

### 결과 값 설명

| 컬럼 | 값 | 설명 |
|---|---|---|
| 예약여부 | 예약 / 미예약 | 분석 기간 내 유효 예약 여부 |
| 회원가입여부 | 신규가입 / 기존가입 / 미가입 | 분석 기간 기준 가입 상태 |
| 쿠폰사용여부 | O / X / (빈값) | 해당 예약에 쿠폰 사용 여부 |

### 파일명 규칙

`{날짜}_{캠페인유형}_예약전환_회원가입_쿠폰사용여부.xlsx`

예: `20260402_쿠폰_보유고객_예약전환_회원가입_쿠폰사용여부.xlsx`

## STEP 5: 결과 전달

1. **결과 파일을 대화 중인 채널에 업로드**한다.
2. **채널에 분석 결과 요약 코멘트**를 함께 남긴다:
   ```
   [분석 완료] 잔여 쿠폰 알림톡 전환 데이터
   - 분석 기간: 2026-04-02 14:00:00 ~ 2026-04-09 23:59:59
   - 총 대상자: 28,766명
   - 예약 전환: 1,523명 (5.3%)
   - 신규가입: 892명 (3.1%)
   - 기존가입: 15,234명 (52.9%)
   - 미가입: 12,640명 (43.9%)
   - 쿠폰 사용: 234건
   - 결과 파일: 20260402_쿠폰_보유고객_예약전환_회원가입_쿠폰사용여부.xlsx
   ```

## 주의사항

1. **전화번호 0 누락 문제**: 엑셀에서 전화번호가 숫자로 저장되면 앞의 0이 사라진다. 대상 파일, 제외 파일 양쪽 모두에서 반드시 정규화를 적용할 것.
2. **배치 처리**: DB 쿼리 시 반드시 100명씩 배치로 처리한다. 한 번에 수만 건을 IN 절에 넣지 않는다.
3. **Replica DB**: 읽기 전용이므로 SELECT만 사용한다.
4. **분석 시작일시의 시간**: 날짜뿐 아니라 발송 시각이 중요하다. 불명확하면 반드시 사용자에게 물어본다.
5. **분석 종료일시**: 날짜만 제공되면 `23:59:59`를 붙인다.
6. **한 고객에 여러 예약**: 예약이 여러 건이면 각 예약별로 별도 행을 생성한다. 원본 데이터가 1행이어도 결과는 N행이 될 수 있다.
7. **다중 시트 처리**: 대상 파일에 시트가 여러 개면 모든 시트를 순차 처리하고 결과를 하나의 파일로 통합한다.
