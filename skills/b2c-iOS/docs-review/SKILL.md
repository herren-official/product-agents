# /b2c-ios-docs-review - 문서 최신화 검토

## 실행 알림
이 스킬이 실행되면 가장 먼저 "[b2c-ios-docs-review] 스킬이 실행되었습니다."를 출력할 것

$ARGUMENTS

## 검사 범위

인자에 따라 검사 범위를 결정:
- `all` (기본값, 인자 없을 때): 전체 검사
- `docs`: `.docs/` 문서만
- `skills`: `.claude/skills/` 스킬만
- `agents`: `.claude/agents/` 에이전트만
- `claude-md`: CLAUDE.md + README.md 교차 검증만

## 검사 모드

두 가지 모드를 지원합니다. 기본 모드는 `detect`입니다.

### detect 모드 (기본값)
불일치/오류/누락을 빠르게 검출. 경로 오류, 모순, 누락 항목 위주로 리포트. 간단한 수정용.

### evaluate 모드
7-기준 품질 평가 프레임워크로 문서의 실용성/고도화 여지까지 분석. 대규모 문서 고도화 작업에 사용.

**7-기준 평가 프레임워크:**

| # | 기준 | 측정 방법 |
|---|---|---|
| 1 | 정확성 (Accuracy) | 파일 경로, 명령어, 설정값이 실제 코드/파일시스템과 일치하는가 |
| 2 | 최신성 (Timeliness) | 최근 변경사항(마이그레이션, 리팩토링, 신규 모듈 추가 등)이 반영되었는가 |
| 3 | 실용성 (Usefulness) | 실제 작업 시 참고 가치가 있는가, 불필요한 단계는 없는가 |
| 4 | 완전성 (Completeness) | 커버해야 할 항목 중 빠진 것이 없는가 |
| 5 | 일관성 (Consistency) | 문서 간 용어/형식/참조 방식이 통일되는가 |
| 6 | 중복성 (Duplication) | 같은 내용이 여러 문서에 반복되는가, 합칠 수 있는 문서가 있는가 |
| 7 | 구조성 (Structure) | 계층/분류가 논리적인가, 탐색 효율이 좋은가 |

각 기준 10점 만점으로 점수화하여 리포트.

## 실행 프로세스

### 1단계: 대상 파일 수집

검사 범위에 따라 대상 파일을 Glob으로 수집:

```
Glob: .docs/**/*.md
Glob: .claude/skills/*/SKILL.md
Glob: .claude/agents/*.md
Glob: CLAUDE.md
Glob: README.md
```

### 2단계: 경로 참조 검증

모든 대상 파일에서 마크다운 링크 및 경로 참조를 추출하고 실제 존재 여부 확인:

**검사 항목:**
- `[text](path)` 형식의 마크다운 링크 경로
- `Read: path` 형식의 Read 지시문 경로
- 백틱 내 파일 경로 참조 (`` `.docs/...` ``)
- 앵커 링크 (`#section-name`)가 실제 헤딩과 일치하는지

**경로 해석 규칙:**
- `.docs/...`: 프로젝트 루트 기준
- `../../../.docs/...`: 스킬 파일 기준 상대 경로 -> 프로젝트 루트의 `.docs/`
- `../../.docs/...`: 에이전트 파일 기준 상대 경로 -> 프로젝트 루트의 `.docs/`
- `./FILE.md`: 같은 디렉토리 내 파일

### 3단계: 내용 일관성 검사

**문서 간 모순 검사:**
- 같은 개념(iOS 버전, 테스트 프레임워크, 커버리지 기준 등)이 여러 문서에서 다르게 기술되었는지
- 커밋 타입 목록이 실제 사용과 일치하는지
- CLAUDE.md의 규칙과 개별 문서의 규칙이 일치하는지

**코드-문서 일치 검사 (선택):**
- 문서에 기술된 enum, struct, protocol이 실제 코드와 일치하는지
- 문서의 디렉토리 트리가 실제 구조와 일치하는지

### 4단계: 누락/고아 검사

**누락 검사:**
- `.docs/`에 존재하지만 CLAUDE.md에서 참조되지 않는 파일
- `.claude/skills/`에 존재하지만 CLAUDE.md 스킬 테이블에 없는 스킬
- `.claude/agents/`에 존재하지만 CLAUDE.md 에이전트 테이블에 없는 에이전트
- README.md 디렉토리 트리에서 누락된 파일

**고아 검사:**
- CLAUDE.md에서 참조하지만 실제로 존재하지 않는 파일
- 스킬 YAML의 skills에 등록되었지만 존재하지 않는 스킬
- 에이전트 YAML의 skills에 등록되었지만 존재하지 않는 스킬

### 5단계: 구조/형식 일관성 검사

**스킬 구조 검사:**
- 모든 스킬이 동일한 YAML 프론트매터 필드를 가지는지
- "실행 알림", "$ARGUMENTS", "실행 프로세스" 섹션 존재 여부
- 에러 처리 섹션 존재 여부
- 참조 문서 섹션 위치/형식 일관성

**에이전트 구조 검사:**
- 모든 에이전트가 N-Phase Work Process 형식을 따르는지
- "Skills and Reference Documents" 테이블 존재 여부
- "Decision-Making Framework" 섹션 존재 여부
- "Quality Assurance" 체크리스트 존재 여부

**공통 검사:**
- 이모지 사용 여부 (프로젝트 규칙 위반, GitHub 라벨 예외)
- Read 지시문 경로 스타일 통일 (프로젝트 루트 기준 `.docs/...` 권장)

### 6단계: 결과 리포트 출력

## 출력 형식

### detect 모드 (기본값)

```
## 문서 검토 결과

### 검사 범위
- 대상: {검사 범위}
- 검사 파일: {N}개

| 심각도 | 건수 | 설명 |

| Medium | N건 | 누락된 참조, 구조 불일치 등 |
| Low | N건 | 형식 불일치, 중복 등 |

### High Priority

- 수정 제안: {구체적 수정 방법}

... (문제별 반복)

### Medium / Low Priority

- 모드: evaluate

| 중복성 | X | ... |

### 발견된 문제
- ...

### 개선 제안
- ...

```

## 에러 처리

| Error | Action |
|-------|--------|
| 파일 읽기 실패 | 해당 파일을 "접근 불가"로 리포트하고 계속 진행 |
| 앵커 링크 검증 불가 | 해당 링크를 "검증 불가"로 리포트 |
| 검사 범위 인자 오류 | 기본값(all)으로 진행 |

## 참조 문서

- CLAUDE.md: 프로젝트 전체 규칙 및 참조 문서 목록
- `.docs/conventions/CONVENTIONS.md`: 코딩 컨벤션
- `.docs/COMMIT_CONVENTION.md`: 커밋 규칙
- `.docs/PR_CONVENTION.md`: PR 규칙
