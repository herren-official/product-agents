## GitHub PR 리뷰 코멘트 (전체 요약 body)

`gh api` reviews 엔드포인트의 `body` 필드에 들어가는 전체 리뷰 요약.

```markdown
## [Claude Code Review]

## 개요

{PR 변경 내용을 1~2문장으로 요약. diff를 분석하여 "무엇을 왜 변경했는지" 서술}

## 변경 사항

| 코호트 / 파일 | 요약 |
|---|---|
| **{변경 그룹명}** | |
| `{파일명1}` | {파일별 변경 요약} |
| `{파일명2}` | {파일별 변경 요약} |

## 리뷰 요약

| 항목 | 내용 |
|------|------|
| 영향 모듈 | `{modules}` (Spring Boot {version}) |
| 변경 규모 | +{additions} / -{deletions} ({files_count} files) |
| 테스트 포함 | {있음 / 없음} |
| Critical | {count}건 |
| Major | {count}건 ({간략 요약}) |
| Suggestion | {count}건 ({간략 요약}) |

### 예상 리뷰 난이도
{level} ({label}) | ~{minutes}분

> 난이도 기준:
> 1 (Trivial) ~5분 | 2 (단순) ~12분 | 3 (보통) ~20분 | 4 (복잡) ~45분 | 5 (매우 복잡) ~1시간+

---
<sub>Generated by [Claude Code](https://claude.ai/claude-code) PR Reviewer</sub>
```

## GitHub PR 인라인 코멘트 (파일/라인별)

`gh api` reviews 엔드포인트의 `comments` 배열에 들어가는 개별 코멘트.

### 형식

모든 인라인 코멘트는 다음 prefix로 시작:

```
**[Claude Code Review]** {심각도 이모지} **{이슈 제목}**
```

### 심각도별 이모지

| 심각도 | 이모지 | 용도 |
|--------|--------|------|
| Critical | 🔴 | 반드시 수정 필요 |
| Major | ⚠️ | 수정 권장 |
| Suggestion | 💡 | 선택적 개선 |

### 인라인 코멘트 예시

```markdown
**[Claude Code Review]** ⚠️ **중복 코드 추출 권장**

이 블록이 N개 파일에 동일하게 반복됩니다.

공통 유틸로 추출하면 유지보수성이 크게 올라갑니다:

\```kotlin
// 제안 코드
\```
```

```markdown
**[Claude Code Review]** 🔴 **NPE 위험**

`customerNo`가 nullable인데 null 체크 없이 접근하고 있습니다.

\```kotlin
// 수정 전
val name = customer.name

// 수정 후
val name = customer?.name ?: throw IllegalArgumentException("고객 정보 없음")
\```
```

## gh api 호출 형식

```bash
gh api repos/herren-official/fineadple-server/pulls/{PR_NUMBER}/reviews \
  -X POST \
  --input - <<'EOF'
{
  "event": "COMMENT",
  "body": "{전체 리뷰 요약 -- 위 템플릿}",
  "comments": [
    {
      "path": "{파일 경로}",
      "line": {라인 번호},
      "body": "**[Claude Code Review]** {심각도} **{제목}**\n\n{상세 설명}"
    }
  ]
}
EOF
```

### 라인 번호 확인

인라인 코멘트 등록 전 반드시 PR 브랜치의 파일에서 정확한 라인 번호를 확인:

```bash
gh api "repos/herren-official/fineadple-server/contents/{file_path}" \
  -H "Accept: application/vnd.github.raw" \
  --method GET -F ref={head_branch} | grep -n "{검색어}"
```

## 터미널 출력 템플릿

터미널에 출력할 때는 GitHub 코멘트와 동일한 구조이되, 더 상세한 정보를 포함합니다:
- 각 이슈에 파일 경로와 라인 번호 포함
- 수정 방법 제안 포함 (코드 예시)
- 전체 체크리스트 결과 테이블 포함

```markdown
## [Claude Code Review] PR #{pr_number}

## 개요
{PR 변경 내용 요약}

## 변경 사항
| 코호트 / 파일 | 요약 |
|---|---|
| `{파일명}` | {요약} |

## 리뷰 요약
| 항목 | 내용 |
|------|------|
| 영향 모듈 | `{modules}` |
| ... | ... |

### 예상 리뷰 난이도
{난이도}

### Critical ({critical_count}건)
{번호}. **[{카테고리}]** `{파일경로}:{라인}` - {설명}
   - 권장: {수정 방법}

### Major ({major_count}건)
{번호}. **[{카테고리}]** `{파일경로}:{라인}` - {설명}
   - 권장: {수정 방법}

### Suggestions ({suggestion_count}건)
{번호}. **[{카테고리}]** {설명}

### 체크리스트 요약
| 카테고리 | 결과 |
|---------|------|
| DDL/마이그레이션 | {N/A or N건 발견} |
| NPE 위험 | {결과} |
| 트랜잭션 경계 | {결과} |
| PII 로깅 | {결과} |
| 네이밍 컨벤션 | {결과} |
| 로깅 | {결과} |
| 아키텍처 | {결과} |
| 테스트 코드 | {결과} |
```
